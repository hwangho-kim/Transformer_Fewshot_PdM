# Transformer_Fewshot_PdM
# 🚀 반도체 양산형 FDC 파운데이션 모델 파이프라인

본 파이프라인은 시계열 데이터의 전처리부터 이상 탐지(Anomaly Detection), 시각화, 원인 분석(RCA)까지 자동화된 완벽한 MLOps(Machine Learning Operations) 아키텍처를 구현합니다.

## 📍 Step 1: 데이터 수집 및 초고속 동기화 (Data Engineering)
장비에서 수집되는 원시(Raw) 데이터가 가진 '불규칙성'을 딥러닝 모델이 소화할 수 있도록 규격화하는 단계입니다.

* **가변 길이 문제 해결 (1D Linear Interpolation):** 실제 반도체 레시피는 장비 상태나 웨이퍼에 따라 공정 소요 시간(예: 40~60스텝)이 미세하게 다릅니다. 기존의 DTW(Dynamic Time Warping) 알고리즘은 연산량이 너무 커서($O(N^2)$) 양산 적용에 무리가 있었습니다. 이를 해결하기 위해 **1D 선형 보간법**을 적용하여, 데이터의 고유한 위상(Phase)은 유지한 채 모든 웨이퍼 데이터를 0.1초 만에 50스텝으로 초고속 동기화합니다.
* **이상 시나리오 주입 및 라벨링:** 검증을 위해 데이터셋의 특정 구간에 '가스 헌팅', '압력 지연', 'RF 파워 글리치'와 같은 명시적 불량 파형과 불량 계측치(Etch Depth)를 주입합니다.
* **데이터 분할 및 정규화:** 전체 1,000장의 데이터를 `Pre-train(800장)`, `Few-shot(20장)`, `Test(180장)`로 나누고, 센서별 스케일 차이를 맞추기 위해 정규화(Standard Scaling)를 수행합니다.

---

## 📍 Step 2: 트랜스포머 오토인코더 설계 (Model Architecture)
다변량 시계열 데이터의 복잡한 패턴을 학습하기 위한 핵심 인공지능 모델 구조입니다.

* **인코더 (Encoder - 모델의 '눈'):** 7개의 센서 데이터를 입력받아 Self-Attention 메커니즘을 통해 시간적 흐름과 센서 간의 상관관계를 파악합니다. 전체 50스텝의 복잡한 파형 정보를 가장 핵심적인 특징만 추려내어 **16차원의 잠재 벡터(Latent Vector)**로 강력하게 압축합니다.
* **디코더 (Decoder - 모델의 '붓'):** 압축된 16차원의 잠재 벡터를 다시 펼쳐서, 원래의 '50스텝 x 7센서' 파형으로 완벽하게 복원(Reconstruction)하는 역할을 수행합니다.

---

## 📍 Step 3: 파운데이션 모델 사전 학습 (Pre-training)
오직 '정상 웨이퍼' 데이터만 사용하여 설비의 기준 동작을 학습하는 단계입니다.

* **자기지도 학습 (Self-Supervised Learning):** 800장의 정상 웨이퍼 데이터를 모델에 입력하고, 모델이 출력한 파형이 실제 입력 파형과 똑같아지도록 훈련(MSE Loss 최소화)시킵니다.
* **골든 궤적(Golden Trace) 체득:** 이 과정을 수십 번 반복하면, 모델은 이 설비의 **'정상적인 동작 기준선(Baseline)'**이 무엇인지 스스로 체득하게 됩니다. 즉, 정상 궤적을 그리는 완벽한 '화가'가 됩니다.

---

## 📍 Step 4: 진단 헤드 분리 및 Few-Shot 강화 (Diagnostic Enhancement)
단 20장의 계측 데이터(정상+불량 섞임)를 활용하여, 모델의 이상 탐지 민감도를 극대화하는 가장 핵심적인 MLOps 설계 패턴입니다.

* **파운데이션 모델 동결 (Freeze):** Step 3에서 완성된 오토인코더의 가중치를 꽁꽁 잠급니다. 이를 통해 향후 불량 데이터를 학습하더라도 **'정상 파형(Expected Trace)'을 예쁘게 그려내는 능력을 까먹는 현상(모드 붕괴, Mode Collapse)을 완벽히 차단**합니다.
* **진단 헤드 (Diagnostic Head) 부착:** 오토인코더 뒤에 센서별 복원 오차(MSE)를 입력으로 받아 최종 이상 점수(Health Index)를 판별하는 소형 신경망(MLP)을 하나 더 달아줍니다.
* **대조 학습 (Contrastive Learning):** 진단 헤드에게 Few-shot 데이터를 보여주며, 정상 웨이퍼면 1점, **불량 웨이퍼(계측치 불량)면 15점의 엄청난 에러 점수를 주도록 강력하게 훈련**시킵니다. 이를 통해 미세한 불량 징후도 놓치지 않고 증폭시켜 탐지해 냅니다.

---

## 📍 Step 5: 동적 제어 차트 및 현업 맞춤형 스코어링 (Evaluation & Control Chart)
학습되지 않은 Test 웨이퍼 180장에 대해 실제 진단을 수행하고, 현업 친화적으로 시각화합니다.

* **동적 제어 한계선 (Dynamic UCL):** 장비는 시간이 지날수록 챔버 상태가 변합니다(Aging). 최근 50장 웨이퍼의 이동 평균(Moving Average)과 표준편차를 활용하여 실시간으로 6-Sigma 한계선을 그어줍니다. 알람 기준이 장비 상태에 맞춰 유연하게 변합니다.
* **100점 만점 환산 스코어링:** 기존의 '에러 비율(작을수록 좋음)'은 직관적이지 않습니다. 이를 현업 엔지니어에게 익숙한 **'100점 만점 스케일(클수록 좋음)'**로 변환합니다. 최대 90점에서 최소 45점으로 매핑하며, UCL 이탈 시점을 **60점(경고 임계점, Warning)**으로 설정하여 붉은색 배경으로 즉각적인 알람을 제공합니다.

---

## 📍 Step 6: XAI 기반 RCA(Root Cause Analysis) 히트맵 분석
불량으로 탐지된 웨이퍼에 대해 '왜 알람이 울렸는지' 근본 원인을 명확하게 짚어냅니다.

* **Original vs Expected 시각화:** 실제 수집된 삐뚤어진 파형(검은색)과, 동결된 모델이 "원래는 이렇게 생겼어야 해"라며 그려낸 이상적인 정상 파형(붉은 점선)을 겹쳐서 보여줍니다.
* **2D RCA Heatmap:** 원본과 복원본의 오차(MSE)를 수치화하여 **[Y축: 7개 센서, X축: 50 스텝]** 형태의 히트맵으로 제공합니다. 이를 통해 **"정확히 어느 공정 스텝에서, 어떤 센서가 튀었는지"** 붉은색으로 한눈에 파악할 수 있습니다.
* **자동 진단 리포트 (Automated Reporting):** 히트맵 결과를 텍스트로 번역하여, "메인 식각 구간(25번째 스텝 부근)에서 C4F8 가스의 오차가 평소 대비 12배 폭증했습니다."와 같은 자연어 리포트를 생성, 엔지니어의 즉각적인 트러블슈팅을 돕습니다.
